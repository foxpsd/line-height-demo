span1.clientHeight = 57
span2.clientHeight = 50
span3.clientHeight = 50

明显看到三者的背景颜色的范围不同。下面进行解释。

首先，font-size 是文字顶线到文字底线之间的距离。line-height 是两行基线之间的距离。那么line-height - font-size = 行距。
当line-height = font-size时，行距等于0，也就是说第一行的底线和第二行的顶线重合。（但不能说两行文字之间没有空隙，因为文字字符大多不会跨越整个font-size的高度）

1.不设置。此时 span 标签的显示高度由内部的文字撑开。内部的文字有 font-size = 50。而 line-height 的值和 font-size 有关系。 chrome 浏览器默认的 line-height = 1.14。也就是说任何一个具体的 inline 或者 inline-block 元素如果没有设置 line-height，那么它的 line-height = font-size * 1.14 = 57。所以 span1 的 line-height = 57。 span1 就被这个 line-height 撑开了，高度是57。
当line-height 改变时，文字的位置不会变化，撑开的高度被平均分配到文字的上下，也就是半行距。

2.设置了height，但没有设置 line-height 时， span 标签撑开的距离就不由计算得到的 line-height 决定了，而是由声明设置的 height决定。当然，各个设置为 inline-block 的 span 标签的文字还是对齐的。此时，背景部分顶部对齐向下延展。

3.设置了line-height。此时 span 标签撑开的高度就是由这个设置的 line-height 决定的，略小于默认计算出来的line-height。
